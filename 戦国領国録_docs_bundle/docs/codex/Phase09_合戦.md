# Phase09 合戦

## 1. 目的

最大5対5部隊のリアルタイム合戦を実装し、兵科・地形・士気・疲労・命令を成立させる。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- BattleScene
- 最大5対5部隊
- 足軽・騎馬・鉄砲
- 地形補正
- 移動・攻撃
- 士気
- 疲労
- 退却
- 勝敗
- 一時停止中の命令
- 結果反映
- 3戦場マップ対応基盤

---

## 5. 実装対象外

- 海戦
- 攻城城内戦
- 武将スキルツリー
- 複雑な物理演算
- 未確定の戦闘数値を独断決定

---

## 6. 実装順序

1. BattleState
2. Unit命令
3. 戦場マップ読込
4. 地形判定
5. 移動
6. 射程・攻撃
7. 損害・士気・疲労
8. 退却
9. 勝敗
10. 全国状態へ反映
11. エフェクトとプール
12. E2E

---

## 7. アーキテクチャ方針

- BattleDomainとBattleSceneを分離
- 戦闘解決は固定tick
- Sceneは入力・描画・エフェクト
- RNGは保存可能な状態を利用
- 結果反映はBattleResultMapperで一括処理

---

## 8. データ・状態管理

- 部隊数最大5対5
- 兵数・士気・疲労は範囲制約
- 地形補正はmaster data
- 兵科相性・数値はGDD承認済みのみ
- 戦闘履歴を記録可能にする

---

## 10. 完了条件

- 最大5対5の合戦が成立する
- 3兵科と地形が反映される
- 士気・疲労・退却が動作する
- 結果が全国状態へ正しく反映される
- 60FPS目標を計測する
- lint/typecheck/test/build/e2eが成功する

## 11. Codexへの禁止事項

- 戦闘計算をSceneへ書く
- パーティクルを毎回生成し続ける
- 兵科数を追加する
- 海戦を実装する
- 数値を推測で決める

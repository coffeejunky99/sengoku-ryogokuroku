# Phase05 武将

## 1. 目的

武将の一覧・詳細・所属・位置・忠誠・俸禄・移動・登用の基盤を実装する。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- 武将マスターデータ
- 武将一覧
- 武将詳細
- 所属・現在地
- 忠誠・俸禄
- 状態管理
- 城間移動
- 登用
- 内政命令への武将割当
- 死亡・負傷・捕虜状態の基盤

---

## 5. 実装対象外

- 複雑な血縁イベント
- 婚姻
- 育成ツリー
- 大規模人事制度
- GDD未定の登用成功率を独断で確定すること

---

## 6. 実装順序

1. GeneralDefinition/State
2. 武将データ検証
3. 武将一覧Query
4. 詳細UI
5. 状態遷移
6. 城間移動
7. 内政割当連携
8. 登用Use Case
9. 忠誠・俸禄UI
10. テスト

---

## 7. アーキテクチャ方針

- 武将状態遷移はDomainへ集約する
- `idle`等のdiscriminated unionを使う
- UIから直接statusを書き換えない
- 同時に複数任務へ就けない不変条件を維持する

---

## 8. データ・状態管理

- 能力値は0〜100
- 現在地は城または移動中の一方
- 在野はclanId=null
- 死亡武将は命令対象外
- 本番武将30名の正式値が未定なら仕様不足として停止する

---

## 9. テスト内容

### Unit
- 状態遷移
- 二重割当拒否
- 死亡・負傷時制約
- 城間移動
- 登用条件

### Integration
- 内政命令割当
- UI詳細
- 所属変更
- 時間進行連携

### E2E
- 武将一覧
- 詳細表示
- 移動
- 内政割当
- 登用操作

---

## 10. 完了条件

- 武将一覧・詳細が動作する
- 所属・位置・状態が整合する
- 二重任務が発生しない
- 移動・登用が仕様通り動く
- lint/typecheck/test/build/e2eが成功する

---

## 11. Codexへの禁止事項

- 仮武将を本番データへ残す
- UIで忠誠や成功率を計算する
- 婚姻や育成ツリーを追加する
- 状態矛盾を型アサーションで隠す

---

## 12. 完了報告形式

1. 問題の整理
2. 実装方針
3. 実装内容
4. 影響範囲
5. テスト内容
6. 完了条件

満たしていない条件がある場合は未完了として報告する。

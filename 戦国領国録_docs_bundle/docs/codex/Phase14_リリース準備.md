# Phase14 リリース準備

## 1. 目的

縦切り試作を配布可能な品質へ整え、総合テスト・性能・互換性・ドキュメント・公開準備を完了する。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- 全機能総合テスト
- リグレッション
- 性能計測
- メモリリーク確認
- セーブ互換性
- iOS/Android/Web build
- エラー処理
- アクセシビリティ
- ドキュメント
- リリースチェックリスト
- 既知問題一覧
- バージョン付与

---

## 5. 実装対象外

- 正式課金
- 広告
- オンライン機能
- 全国版コンテンツ
- 未承認の追加機能
- GDD外の演出追加

---

## 6. 実装順序

1. 仕様差分監査
2. 全テスト
3. E2E
4. セーブ移行
5. 性能
6. メモリ
7. 実機
8. アクセシビリティ
9. エラー文言
10. ドキュメント
11. version
12. release candidate
13. 最終受入確認

---

## 7. アーキテクチャ方針

- 新機能を追加しない
- バグ修正と品質改善に限定
- 変更ごとに回帰テスト
- 仕様差分はGDDへ照合
- リリース候補を固定して検証する

---

## 8. データ・状態管理

- 本番データに仮値がない
- ID重複がない
- 参照整合性が通る
- セーブ移行が全対応版で通る
- マスターデータversionを記録する

---

## 9. テスト内容

### Automated
- lint
- typecheck
- unit
- integration
- build
- e2e
- visual
- migration
- repository contract

### Manual
- Web
- iOS
- Android
- 新規開始
- 勝利
- 敗北
- 保存・読込
- 長時間プレイ
- バックグラウンド復帰
- 低性能端末確認

---

## 10. 完了条件

- GDDの縦切り試作範囲を満たす
- 重大バグ0
- 型・lint・test・build・e2e成功
- Web/iOS/Android build成功
- セーブ互換性確認
- 60FPS目標・最低30FPS確認
- 既知問題一覧作成
- リリースチェックリスト完了
- バージョン付与
- 受入テスト合格

---

## 11. Codexへの禁止事項

- リリース直前に新機能を追加する
- 既知の重大バグを隠す
- テスト失敗を無視する
- 仮データを残す
- デバッグログを残す
- 互換性を壊す変更をmigrationなしで行う

---

## 12. 完了報告形式

1. 問題の整理
2. 実装方針
3. 実装内容
4. 影響範囲
5. テスト内容
6. 完了条件

満たしていない条件がある場合は未完了として報告する。

# Phase07 軍団と行軍

## 1. 目的

武将最大5名の軍団編成、経路探索、行軍、補給、全国マップ表示を実装する。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- 軍団作成・解散
- 総大将設定
- 最大5武将
- 兵科・兵数
- 経路探索
- 出兵
- 行軍進捗
- 軍団資金
- 補給消費
- 全国マップ上の軍団表示
- 到着処理

---

## 5. 実装対象外

- 攻城解決
- 合戦
- 海上移動
- 水軍
- AI出兵
- 未確定の移動速度・補給量の独断決定

---

## 6. 実装順序

1. ArmyState/UnitState
2. 軍団編成Use Case
3. 不変条件
4. 街道グラフ
5. 経路探索
6. 行軍状態
7. tick更新
8. 補給消費
9. Map DTO連携
10. 到着イベント

---

## 7. アーキテクチャ方針

- 経路探索はDomain service
- 軍団表示はPhaser
- 行軍状態はGameState
- 軍団位置を城所属と移動中で排他的に管理
- 兵科定義はmaster data

---

## 8. データ・状態管理

- 武将は複数軍団へ所属不可
- 総大将は軍団内武将
- 最大5名
- 兵数・資金は負数不可
- 経路が存在しない場合は出兵不可
- 速度・消費量は承認済みデータのみ

---

## 9. テスト内容

### Unit
- 最大5名
- 二重所属拒否
- 総大将整合性
- 最短経路
- 移動進捗
- 補給不足
- 到着

### Integration
- 軍団編成UI
- 出兵Command
- Map表示
- 時間進行連携

### E2E
- 軍団編成
- 目的地選択
- 行軍開始
- マップ移動
- 到着

---

## 10. 完了条件

- 軍団編成が成立する
- 経路探索が正しい
- 行軍が時間進行で進む
- 軍団がマップへ表示される
- 補給状態が更新される
- lint/typecheck/test/build/e2eが成功する

---

## 11. Codexへの禁止事項

- 攻城や合戦を先行実装する
- 経路をコードへ直書きする
- Phaser座標を権威状態にする
- 5名上限を無視する

---

## 12. 完了報告形式

1. 問題の整理
2. 実装方針
3. 実装内容
4. 影響範囲
5. テスト内容
6. 完了条件

満たしていない条件がある場合は未完了として報告する。

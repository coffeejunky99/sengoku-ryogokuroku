# Phase08 攻城戦

## 1. 目的

敵城到着後の包囲・強攻・降伏勧告・落城処理を実装する。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- 攻城開始
- 包囲
- 強攻
- 降伏勧告
- 守備兵・防御値
- 士気・補給との連携
- 落城
- 城所有者変更
- 城内武将処理
- 自動停止
- 攻城UI

---

## 5. 実装対象外

- 戦場マップでの野戦
- 詳細な城内戦闘
- 海上封鎖
- AI攻城判断
- 未確定の損害式・降伏率の独断決定

---

## 6. 実装順序

1. SiegeState
2. 攻城開始条件
3. 包囲進捗
4. 強攻解決
5. 降伏判定
6. 落城トランザクション
7. 武将・兵・施設状態更新
8. UI
9. 通知
10. テスト

---

## 7. アーキテクチャ方針

- 攻城は状態機械で管理
- 落城処理は一括更新し中間不整合を残さない
- UIは選択肢と予測を表示
- 計算はDomainへ置く
- Phaserは包囲表示のみ

---

## 8. データ・状態管理

- 城所有者変更時に城主・駐留武将を整合させる
- 守備兵は負数にしない
- 攻城中軍団は他行動不可
- 降伏条件はGDD承認値のみ
- 落城履歴を記録する

---

## 9. テスト内容

### Unit
- 攻城開始条件
- 包囲進捗
- 強攻損害
- 降伏可否
- 落城状態更新
- 同時攻城競合

### Integration
- 行軍到着から攻城開始
- 城所有者更新
- 自動停止
- UI表示

### E2E
- 敵城到着
- 包囲
- 強攻
- 降伏勧告
- 落城

---

## 10. 完了条件

- 3種類の攻城行動が動作する
- 落城時の状態が整合する
- 競合状態を処理できる
- 自動停止と通知が動作する
- lint/typecheck/test/build/e2eが成功する

---

## 11. Codexへの禁止事項

- 野戦合戦を先行実装する
- 落城処理を複数箇所へ重複実装する
- 未確定損害式を決める
- 城所有者だけ変更して武将状態を放置する

---

## 12. 完了報告形式

1. 問題の整理
2. 実装方針
3. 実装内容
4. 影響範囲
5. テスト内容
6. 完了条件

満たしていない条件がある場合は未完了として報告する。

# Phase02 全国マップ

## 1. 目的

全国マップの描画基盤を実装し、城・街道・勢力色・選択・カメラ操作を成立させる。ゲームロジックはまだ最小限に留める。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

6. カメラパン・ズーム
7. 城選択優先度
8. Bridgeイベント
9. React側選択表示
10. スマートフォンviewport確認

---

## 7. アーキテクチャ方針

- Phaserは描画と入力だけを担当する
- 城所有者や勢力情報はGameStateからDTOへ変換する
- SceneからZustand内部を直接更新しない
- Bridgeで`castle-selected`を通知する
- 表示オブジェクトは差分更新する
- カメラ処理は独立Systemへ分離する

---

## 8. データ・状態管理

- 城・街道はJSONから読み込む
- ID参照整合性を検証する
- 城座標はマスターデータで管理する
- 表示名をIDとして使用しない
- 城選択状態はuiStoreへ保持する
- Phaserオブジェクトへ権威的状態を保存しない

---

## 9. テスト内容

### Unit
- 城・街道データ検証
- DTO変換
- カメラ境界計算
- 選択優先度

### Integration
- Bridgeで城IDがReactへ届く
- Store更新で対象城だけ描画更新される

### E2E
- 10城が表示される
- 城をタップして選択できる
- パン・ズームできる
- スマートフォンで横スクロールしない
- canvasが1つだけ存在する

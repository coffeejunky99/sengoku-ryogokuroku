# Phase04 城と内政

## 1. 目的

城詳細画面と内政命令の開始・進行・完了までを実装する。

---

## 2. 参照文書

実装前に以下を読む。

- `AGENTS.md`
- `docs/codex/README.md`
- `docs/GDD/GDD.md`
- `docs/design/data-design.md`
- `docs/design/technical-design.md`
- 直前Phaseの完了報告

GDDに存在しない仕様は実装しない。

---

## 3. 開始条件

- 直前Phaseが完了している
- `npm run lint` が成功する
- `npm run typecheck` が成功する
- `npm run test` が成功する
- `npm run build` が成功する
- 未解決の重大エラーがない
- 対象機能に必要な仕様がGDDに存在する

条件を満たさない場合は開始しない。

---

## 4. 実装対象

- 城詳細UI
- 城状態表示
- 農業開発
- 商業振興
- 救恤
- 治安維持
- 城郭強化
- 軍事訓練
- 兵士募集
- 施設建設
- 命令プレビュー
- 費用・所要時間・効果表示
- 命令進捗

---

## 5. 実装対象外

- 武将移動
- 外交
- 軍団
- 攻城
- AI
- GDD未確定の効果量・費用を本番値として確定すること

---

## 6. 実装順序

1. CastleState実装
2. DomesticTask型
3. 命令可否判定
4. プレビューQuery
5. 開始Use Case
6. 時間進行との連携
7. 完了効果適用
8. 城詳細React UI
9. 通知
10. 境界値テスト

---

## 7. アーキテクチャ方針

- 計算式はDomain serviceへ置く
- UIはPreview QueryとCommandを利用する
- 城詳細コンポーネントへ計算式を書かない
- 命令状態は正規化Recordで管理する
- 完了処理はsimulation tick後の安全なタイミングで行う

---

## 8. データ・状態管理

- 城初期値はシナリオデータ
- 施設定義はmaster JSON
- 効果量・費用・時間は承認済み値のみ使用
- 未確定値はテストfixtureへ限定
- 城能力値はcapacityを超えない
- 数量は負数にしない

---

## 9. テスト内容

### Unit
- 命令可否
- 費用不足
- 能力上限
- 完了時効果
- キャンセル処理
- 施設枠上限

### Integration
- UIプレビューと実行結果一致
- 時間進行で完了
- Store更新
- 通知

### E2E
- 城選択
- 命令選択
- 確認
- 進捗
- 完了反映

---

## 10. 完了条件

- 城詳細が表示される
- 内政命令を開始できる
- 費用・時間・予測効果を確認できる
- 完了時に状態が更新される
- 不正命令を拒否できる
- lint/typecheck/test/build/e2eが成功する

---

## 11. Codexへの禁止事項

- 効果量をUIへ直書きする
- GDD未定値を適当に決める
- 城状態をPhaserへ保存する
- 武将機能を先行実装する

---

## 12. 完了報告形式

1. 問題の整理
2. 実装方針
3. 実装内容
4. 影響範囲
5. テスト内容
6. 完了条件

満たしていない条件がある場合は未完了として報告する。
